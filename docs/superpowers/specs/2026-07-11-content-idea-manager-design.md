# עיצוב: אפליקציית ניהול רעיונות לתוכן (מאיה קיסלב)

תאריך: 2026-07-11

## רקע ומטרה

מאיה מריצה מדי שבוע תהליך מחקר רעיונות (skill: `weekly-content-ideas`) שמייצר קובץ נפרד עם עשרות רעיונות מסווגים ל-4 סוגי תוכן (ערכי/אישי/מכירתי/בידורי). כרגע אין מקום מרכזי אחד לראות את כל הרעיונות יחד, לעקוב אחרי הסטטוס שלהם (מה עוד לא בוצע מול מה כבר פורסם), ולחפש רעיון ישן לפי נושא. האפליקציה הזו פותרת את זה.

**מה חשוב לה (מהשיחה):**
- מעקב סטטוס - לראות את כל הרעיונות במקום אחד, ולסמן לכל אחד: טרם בוצע / בתהליך / פורסם.
- מאגר לחיפוש והשראה - למצוא רעיון ישן לפי קטגוריה/תאריך/מקור.
- כניסת רעיונות היא **ידנית בלבד** - היא בעצמה מחליטה אילו רעיונות מהסקירה השבועית שווים להיכנס לאפליקציה (תואם את משמעת הליבה שלה לגבי אימות רעיונות - ראו זיכרון `maya-kislev-working-style`).
- גישה גם מהנייד וגם מהמחשב, עם סנכרון בין המכשירים.

## ארכיטקטורה

- **Frontend:** אפליקציית web עצמאית (Vanilla HTML/CSS/JS, ללא build step, ללא framework) - קובץ `index.html` + קבצי JS/CSS נלווים.
- **Hosting:** GitHub Pages, באותו ריפו/דפוס כמו האתרים הקיימים שלה (ראו זיכרון `hosting-migration-github-pages`).
- **Backend/נתונים:** Firebase Firestore - מסד נתונים בענן, בשכבה החינמית (Spark plan), מספיק בהרבה סדרי גודל למספר רעיונות שהיא צפויה לצבור.
- **אימות:** Firebase Authentication עם ספק Google (כניסה חד-פעמית עם חשבון ה-Google שלה, נשארת מחוברת אח"כ בדפדפן).
- **הרשאות נתונים:** Firestore Security Rules יגבילו קריאה/כתיבה למשתמש המחובר (ה-`uid` שלה) בלבד - שכבת אבטחה בסיסית למניעת גישה זרה, גם שהאפליקציה אינה חושפת נתונים רגישים.

## מודל נתונים

קולקציית Firestore בודדת: `ideas`, כל מסמך:

| שדה | סוג | חובה/אופציונלי | הערות |
|---|---|---|---|
| `title` | טקסט | חובה | כותרת קצרה לרעיון |
| `category` | enum | חובה | אחד מ: `ערכי` / `אישי` / `מכירתי` / `בידורי` |
| `hookText` | טקסט ארוך | חובה | טקסט ה-hook המוכן להעתקה |
| `status` | enum | חובה, ברירת מחדל `רעיון` | אחד מ: `רעיון` / `מתוכנן` / `צולם/הוקלט` / `פורסם` |
| `sourceLink` | URL | אופציונלי | לינק למקור ההשראה |
| `notes` | טקסט ארוך | אופציונלי | הערות/טודו חופשי |
| `createdAt` | timestamp | אוטומטי | לצורך מיון/היסטוריה |
| `updatedAt` | timestamp | אוטומטי | מתעדכן בכל שינוי |

## מסכים ותצוגות

### 1. לוח (מסך ראשי, קנבן)

- 4 עמודות אופקיות: **רעיון → מתוכנן → צולם/הוקלט → פורסם**.
- כל רעיון מוצג ככרטיס בעמודה המתאימה לסטטוס שלו, צבוע בצבע קטגוריה (4 צבעים בולטים וסאטורטיביים, תואם לסגנון "הלוח הדיגיטלי" הצבעוני שאושר בפרויקטים קודמים - לא מינימליזם מרוסן).
- על כל כרטיס: כותרת, תג קטגוריה צבעוני, וחיצי קדימה/אחורה (◀ ▶) למעבר לעמודת סטטוס סמוכה בנגיעה אחת - **לא** גרירה (drag-and-drop), לפי בקשתה המפורשת, כדי שיעבוד אמין גם במגע בנייד.
- נגיעה על גוף הכרטיס (לא על החיצים) פותחת אותו לעריכה מלאה.
- כפתור צף "+ הוסיפי רעיון" (פינה קבועה, נגיש בגלילה) פותח טופס הוספה.

### 2. מאגר/חיפוש

- תצוגת רשימה של **כל** הרעיונות (כולל שפורסמו כבר), עם:
  - שדה חיפוש טקסט חופשי (מחפש בכותרת + hook).
  - סינון לפי קטגוריה (4 הסוגים).
  - סינון לפי סטטוס.
- מטרת המסך: "איפה הרעיון הישן על X שכתבתי לפני חודשיים" - לא רק מעקב, גם ארכיון בר-חיפוש.

### טופס הוספה/עריכה

שדות: כותרת (טקסט קצר, חובה), קטגוריה (בחירה מ-4, חובה), טקסט hook (textarea, חובה), לינק מקור (טקסט, אופציונלי), הערות/טודו (textarea, אופציונלי). בעריכה מתווסף גם שינוי סטטוס ידני (dropdown) וכפתור מחיקה עם אישור.

## זרימת נתונים

1. היא נכנסת לכתובת האפליקציה (מסומנת ב-bookmark/מסך הבית בנייד ובמחשב) → נדרשת כניסה עם Google בפעם הראשונה בלבד בכל מכשיר.
2. רואה את הלוח עם כל הרעיונות הפתוחים (לא פורסמו) ממוינים לפי עמודת סטטוס.
3. מוסיפה רעיון חדש → נשמר ב-Firestore → מופיע מיידית בעמודת "רעיון" בכל המכשירים המחוברים (סנכרון בזמן אמת דרך Firestore listeners).
4. לוחצת על חץ בכרטיס → הסטטוס מתעדכן ב-Firestore → הכרטיס עובר עמודה.
5. עוברת למסך מאגר/חיפוש → מחפשת/מסננת מתוך כל הרעיונות ההיסטוריים.

## טיפול בשגיאות

- אין חיבור רשת: Firestore offline persistence מופעל - קריאה/הוספה/עריכה ממשיכות לעבוד באופן מקומי ומסתנכרנות אוטומטית כשהחיבור חוזר. הודעת "toast" קטנה מציגה מצב "לא מקוון" כדי שלא תופתע אם השינוי לא מופיע אצלה בנייד עדיין.
- כשל בכניסה (Google auth נכשל): מסך שגיאה פשוט עם כפתור "נסי שוב".
- מחיקת רעיון: דורשת אישור (דיאלוג "בטוחה שאת רוצה למחוק?") כדי למנוע מחיקה בטעות בנגיעה.

## עיצוב חזותי

ממשיך את שפת העיצוב שאושרה בפרויקטים קודמים שלה (ראו זיכרון `maya-kislev-working-style`): רקע כהה, כרטיסים בצבעי מילוי מלאים ("solid-fill") לפי קטגוריה, לא פסטלים/גבולות דקים. 4 צבעי קטגוריה בולטים ושונים בבירור זה מזה. עיצוב רספונסיבי מלא - נבדק גם בפריסת מובייל צרה (עמודות הופכות לגלילה אופקית או מוצגות אחת מתחת לשנייה במסך צר, לפי מה שיעבוד טוב יותר בפועל).

## בדיקה

בדיקה ידנית (לא נדרשת חבילת טסטים אוטומטית לאפליקציה בגודל הזה): הוספה/עריכה/מחיקה/שינוי סטטוס/חיפוש/סינון, בדסקטופ ובמובייל (Chrome + Safari), כולל תרחיש "שני מכשירים פתוחים במקביל" לוודא שהסנכרון עובד בזמן אמת.

## מחוץ לתחום (לא בגרסה הזו)

- ייבוא אוטומטי מהקובץ השבועי - הוחלט במפורש שלא, היא רוצה שליטה ידנית על מה שנכנס.
- ריבוי משתמשים/הרשאות לתלמידים - האפליקציה למשתמש יחיד (מאיה) בלבד.
